## 6. Key Findings

**Headline:** Discoverability predicts actionability (ρ=0.450) but NOT diagnosis (ρ=0.043). Agents diagnose problems equally well without docs — but they can't provide actionable fixes without them.

1. **Diagnosis is doc-independent.** Coverage=None cases actually score *higher* on diagnosis (2.45) than Full (2.16). Agents extract diagnostic signal from the issue context (error messages, stack traces) — docs aren't needed for understanding what went wrong.

2. **Actionability is doc-dependent.** Full coverage → 0.86 mean actionability vs None → 0.12 (Δ=+0.74). Direct discoverability → 0.73 vs Missing → 0.06 (Δ=+0.67). Without docs, agents can name the problem but can't tell users how to fix it.

3. **Partial coverage is the fabrication danger zone.** 5.4% fab rate for Partial vs 0% for Full and 2.3% for None. Partial docs give agents enough context to hallucinate plausible-sounding extensions. Complete docs prevent fabrication; missing docs trigger honest "I don't know."

4. **J3 (Correctness) is the critical gap.** 30% fabrication rate, zero Full coverage, 8 None. This is the highest-risk journey for doc improvement — agents are actively harmful here.

5. **J8 (Custom Ops) has zero actionability** despite 5 Full coverage cases. Docs exist but don't contain actionable fixes — a doc quality problem, not a discoverability problem.

**Implication for improvement strategy:**
- Discoverability improvements (SEO, cross-linking) will improve actionability but not diagnosis.
- Doc quality improvements (deeper content, worked examples) are needed for J8 and other "docs exist but don't help" cases.
- J3 needs new documentation urgently — agents are fabricating answers in the absence of docs.
- Partial → Full coverage upgrades will reduce fabrication, not just improve actionability.
